创建语句

数据库的创建

语法：create database 数据库名字;
例子:
create database shop;

表的创建

语法: create table 表名
(<列名><数据类型><约束>
<列名><数据类型><约束>…
)
例子:
create table Product
(product_id char(4) not null,
product_name varchar(100) not null,
product_type varchar(32) not null,
product_price integer not null,
product_date date ,
primary key(product_id)
)

删除和更新

删除数据库

语法：
drop database 数据库名；
例子：
drop database shop;

删除数据表

语法:
drop table 数据表名字;
例子：
drop table Product；

修改表名

语法:
alter table 表名 rename to 新的表名;
例子:
alter table product1 rename to product2;

更新数据表

添加列

语法:
alter table <表名> add column<列的定义>;
例子：
alter table product add column product_name_pinyin varchar(100)；

删除列

语法：
alter table product drop column<列名>
例子：
alter table product drop column product_name_pinyin；

插入数据

语法:
insert into <表名> values(数据)
例子：
insert into product values(‘0001’,’T恤衫’,’衣服’,’1000’,’2009-09-20’);
insert into product values(‘0002’,’打孔器’,’办公用品’,’400’,’2009-09-11’);

group by常见错误

  1. 在select子句中写了多余的列（select子句中只能是常数,聚合函数,group by子句中指定的列名（聚合键））
    例子：
    select product_name,purchase_price,count(*)
    from product
    group by purchase_price
    在select子句中书写聚合键之外的列名
  2. 在group by中写了列的别名
    例子：
    select product_type as pt, count(*)
    from product
    group by pt;
    错误原因:group by中的列名必须使用原列名不能使用别名
  3. group by 子句结果排序
    group by 结果是随机的，没有进行排序
  4. 在where子句中使用聚合函数
    group by 必须出现在where后

    having为group by聚合结果指定条件

  5. having子句
    select <列名1><列名2>
    from 表名
    where 条件
    group by <列名1><列名2>
    having <分组条件>
    注意：having条件一定要在group by 之后

    order by 对查询结果进行排序

    select <列名1><列名2><列名3>
    from 表名
    order by <排序基准1>
    注意：order by 语句可以使用别名排序

    desc关键字降序默认升序

    子句顺序

    select —–> from ——>where——->group by ——->having ——->order by

    事务

    什么是事务

    事务就是需要在同一批处理单元中执行的一系列更新处理的集合
    例子：如果需要同时更新运动T恤销售单价和T恤衫的销售单价
    分析：两个语句要同时执行，一定要使用事务来进行处理
    语法：
    事务开始语句（start transaction）
    DML语句1；
    DML语句2；
    DML语句3
    事务结束语句（commit或者rollback）
    例子：
    start transaction;
    update product
     set sale_price = sale_price - 1000
    where product_name = ‘运动T恤’;
    update product
     set sale_price = sale_price + 1000
    where product_name = ‘T恤衫’;
    commit;

    复杂查询

    视图

    视图和表

    视图：从sql的角度来看视图就是一张表
    视图和表的区别：表中保存了实际的数据而视图没有保存真实的数据，视图保存的只是select语句
    语法：
    create view 视图名字
    as
    select 语句
    例子：
    create view productsum(product_type,cnt_product)
    as
    select product_type,count(*)
    from product
    group by product_type;
    使用视图
    select product_type,cnt_product
    from productsum;

    视图的限制

  9. 定义视图时不能使用order by 子句
  10. 视图的更新
    只有在视图中满足如下条件才能更新:
     - select子句中未使用distinct
 - from 子句中只有一张表
 - 未使用group by 子句
 - 未使用having 子句

    删除视图

    drop view 视图名字（<列名1>,<列名2>,<列名3>…）

    子查询

    子查询

    子查询概括起来就是一张一次性视图
    例子：
    实现和上面视图一样功能的子查询
    select product_type,cnt_product
    from (select product_type,count(*) as cnt_product
    from product
    group by product_type) as productsum;
    原则上子查询必须设定名称,如上子查询名称为productsum
    子查询执行顺序：先执行内层的select语句再执行外层select语句

    标量子查询

    什么是标量子查询

    标量子查询就是每次查询都必须且只能返回1行1列的结果
    例子：
    需要查询出销售单价高于平均销售单价的商品
    错误方法：
    select product_id.product_name,sale_price
    from product
    where sale_price>avg(sale_price)；
    错误原因：avg聚合函数只能使用在select语句之中，不能用在where之中
    解决方法：标量子查询
    select product_id,product_name,sale_price
    from product
    where sale_price >(select avg(sale_price)from product);

    标量子查询书写位置

    标量子查询不只是使用在where语句之中，可以用在任何语句之中
    例子：标量子查询使用在select之中
    select product_id,
    product_name,
    sale_price,
    (select avg(sale_price)from product as avg_price)
    from product;
    标量子查询使用在haing语句之中
    select product_type,avg(sale_price)
    from product
    group by product_type
    having avg(sale_price)>(select avg(sale_price) from product

    关联子查询

    普通的子查询和关联子查询的区别

    解决的问题：
    选取出各商品种类中高于该商品种类的平均销售单价的商品
    错误方法：使用标量关联子查询
    select product_id,product_name,sale_price
    from product
    where sale_price > (select avg(sale_price)from product group by product_type)
    错误原因：标量关联子查询中的语句并不是标量而是一些组合
    这个时候就需要使用关联子查询
    select product_id,product_name,sale_price
    from product as p1
    where sale_price > (select avg(sale_price) from product as p2
    where pa.product_type = p2.product_type
    group by product_type
    );

    子查询总结

  11. 什么时候需要使用子查询？
    在需要查询语句嵌套的时候需要使用到子查询
  12. 什么时候需要使用标量子查询
    当需要将聚合函数使用在where之中的时候 需要使用子查询
    原因：聚合函数不能使用在where语句之中
  13. 什么时候需要使用关联子查询
    在需要使用子查询的同时需要使用group by语句的时候，这个时候需要使用关联子查询

    什么是集合运算

  34. 表的加法union
    例子：
    select product_id,product_name
    from product
    union
    select product_id,product_name
    from product2;
    注意事项:
    1. 作为运算对象的记录的列数必须相同
      select product_id,product_name
      from product
      union
      select product_id,product_name,sale_price
      from product2
      错误原因：前后列数不一致
    2. 作为运算对象的记录中列的类型必须一致
      select product_id,product_name
      from product
      union
      select product_id,product_name,regist_date
      from product2
    3. 可以使用任何select语句，但order by子句只能在最后一次使用一次
    4. union运算通常都会进行去重操作
  35. 包含重复行的集合运算——-all选项
